新 QC 7つ道具

言語データ（定性データ）を構造化・可視化するツール群。
企画・設計・問題解決の上流工程で使う。数値では表せない問題の整理に強い。

定性データ 上流工程 企画・設計段階 DMAIC-D/A
N1
親和図法（KJ法）
Affinity Diagram
D:Define

バラバラな言語情報（アイデア・意見・問題）を親和性で グループ化し、問題構造を浮き彫りにする。VOC整理に最適。

いつ使う問題の全体像把握・VOC整理・ブレインストーミング後
進め方① カード（付箋）に1事象1カード記入 → ② 親和性でグループ化 → ③ グループ表題付け
親和図 ─ 付箋ボード（クリックで追加）
実施チェック
N2
連関図法
Relations Diagram
A:Analyze

複数の原因・結果が複雑に絡み合う問題を矢印で表現。「出る矢印が多い」→ 根本原因、「入る矢印が多い」→ 重要結果。

いつ使う慢性的・複合的問題の原因構造把握
読み方出矢印数が多いカード＝重点改善対象（根本原因）
連関図 サンプル（矢印＝因果関係）
分析チェック
N3
系統図法
Tree Diagram / Systematic Diagram
M:Method（ARMS）

目標達成のための手段をツリー状に展開。「手段→さらに手段」を繰り返して実行可能レベルまで落とし込む。HOWツリーの実務版。

いつ使う是正処置の立案・設計要求の展開・WBS作成
展開の深さ「誰かがすぐ実行できる」レベルまで展開する
系統図 サンプル（不良率低減）
🔵 末端ノード = 実行可能なアクション（担当者・期限付き）
展開チェック
N4
マトリックス図法
Matrix Diagram
A:Analyze / M

2つの要素群の関係の強さを行列で表現。品質特性と工程の関係、要求と手段の対応などを一覧で整理する。QFD（品質機能展開）の基盤。

いつ使うCTQと工程の関係整理・FMEA・設計QFD
記号◎強い関係 / ○関係あり / △弱い関係 / （空白）無関係
マトリックス図（セルをクリックして関係を設定）
工程 ↓ / CTQ → 寸法精度 表面粗さ 硬度 外観
活用チェック
N5
マトリックスデータ解析法
Matrix Data Analysis (PCA)
A:Analyze

主成分分析（PCA）によりマトリックスデータを2次元に圧縮・可視化。新QC7つ道具の中で唯一の数値解析手法。

いつ使う多変数データの構造把握・製品群の分類・類似性分析
実務での代用ExcelのPCA・MiniTab・JMPなどで実施するケースが多い
IATF文脈MSA・工程能力の多変量分析に応用される
概念図：主成分分析のイメージ
適用チェック
N6
PDPC法
Process Decision Program Chart
I/C:Control

計画実施中に起こりうる「もしもの事態」を事前に列挙し、対応策（Plan B）を決めておく。KT法の潜在問題分析（PPA）と同じ思想。

いつ使う新工程立上げ・重要プロジェクトの事前リスク対策
他との違い「失敗パスと回避策」を同時に図示するのが特徴
PDPC 概念図
作成チェック
N7
アロー図法（日程計画）
Arrow Diagram (PERT/CPM)
Plan:計画立案

作業の順序・依存関係を矢印で表現し、クリティカルパス（最長経路）を特定。プロジェクトの最短完了日と遅延しては困るタスクを明確化。

いつ使う設備立上げ・量産準備・認証取得計画
クリティカルパス赤線＝1日でも遅れるとプロジェクト全体が遅延するタスク

QC7 vs 新QC7 ─ 使い分けガイド
▸ QC 7つ道具 を使う場面
✓ 不良件数・測定値などの数値データがある
✓ 現場の日常管理・工程監視に使いたい
✓ 「何が起きているか」「どの程度か」を把握したい
✓ DMAIC の M（Measure）・A（Analyze）・C（Control）フェーズ
▸ 新QC 7つ道具 を使う場面
✓ 意見・アイデア・問題記述などの言語データしかない
✓ 上流（企画・設計）で問題を構造化したい
✓ 複雑に絡み合う問題を整理したい（連関図）
✓ DMAIC の D（Define）・A（Analyze）・I（Improve）フェーズ
